Deferred — transition signal later

Not in the first pass. v1 simply changes sections at the boundary with no cue — good enough to ship. The cue is a polish layer to add once the core multi-section flow is proven. The design is captured here so we don't lose it.

When a section is about to change, the player benefits from a heads-up to move their hands (or anticipate the new chords). Matters most in Manual mode and with the foot pedal; also makes Auto playback feel musical instead of abrupt.

Layered, each toggleable

1. Visual default on

On the last bar before a change, the next scrubber segment glows amber ("up next") and the beat dots tint amber — a peripheral "get ready" you can catch without reading.

2. Count-in click default on

An accented click on the remaining beats of the last bar — a count-in feel, distinct from the kit so it reads as a cue. Cheap to build, style-independent.

3. Drum fill phase 2

Swap the last bar's drum pattern for a style-appropriate fill (snare/tom roll into the downbeat). The musician-native signal — but needs per-style fill patterns + a song-aware drum engine. Defer.

Timing

  • Lead time = the last bar of the current section. The change lands on the next downbeat.
  • Short sections: a 1-bar section can't give a full bar of lead — degrade to a half-bar / last-two-beats cue. Never longer than the section.

Manual vs Auto

  • Manual: the cue fires only when a change is actually pending (you tapped Next / queued a jump). Tap during the last bar (too late for a full cue) → switch cleanly with no fill; don't delay.
  • Auto: every advance is known, so cue the last bar of each section — but only when advancing to a different section number (skip cues on a repeat like 1 1). Toggle off if it feels busy.
  • The cue always targets the actual next section — sequential next, or the queued-jump target if one is set.

Engine implications

  • Drums become song-aware (for fills) — build a song-length step array with fills on transition bars, mirroring how buildBass already concatenates per-section steps. Today drums loop a 1-bar pattern independent of structure.
  • Count-in click: a small dedicated click/synth triggered on the last bar's beats; no drum-engine rework needed for this part.
  • Visual: the Draw callbacks already fire per bar/beat — add an "is last bar / next section" flag to drive the amber scrubber pulse + beat-dot tint.

Runner-up layout — B (unified timeline) not chosen

Strong second choice — conceptually elegant (one row that is the timeline, never two rows of numbers). Set aside because it moves further from today's layout than D, and D's seek-bar metaphor sells tap-to-jump more naturally. Kept for comparison.

Layout B — one unified timeline

Kill the separate "tabs" row. The single connected pill is the arrangement timeline. When the arrangement is empty, it shows the implicit linear list of all sections (1 2 3 …), played in order. Only ever one row of numbers on screen.
